Das Layout

Jede Seite braucht Struktur

Wie schon in Html Struktur angesprochen braucht jede Seite eine vernünftige Struktur. Bei CSS geht es aber eher um das Aussehen des Layouts, denn der Aufbau wird ja mit HTML festgelegt. Damals wurden oft Tabellen genutzt um die Seite zu strukturieren, jedoch sollte man tags nicht zweckentfremden und deswegen wird nun meist eines der unten genannten Layouts genutzt.

Das "float-layout"

Dies ist das "Standard" Layout jeder Seite. Die Funktionsweise ist recht einfach, alle Elemente "fließen" von links nach rechts und von oben nach unten. Das einfache an diesem Layout ist, es funktioniert auch ohne das etwas speziell mit CSS verändert wird. Natürlich gibt es aber für besondere Ansprüche auch Eigenschaften welche man verändern kann um das gewünschte Aussehen zu erhalten. Die folgenden Eigenschaften können auf jedem Element angewendet werden.

welche Eigenschaften werden gebraucht?

  • float:
    • Welche Werte bewirken was?
    • right; Das Element fließt auf der Rechten Seite weiter
    • left; Das Element fließt auf der Linken Seite weiter
    • none; Das Element fließt genau da wo es auch im html-code steht weiter
  • clear:
    • Welche Werte bewirken was?
    • right; Das Element kann NICHT von rechts-fließendes Elementen überdeckt werden
    • left; Das Element kann NICHT von links-fließendes Elementen überdeckt werden
    • both; Das Element kann von KEINER Seite überdeckt werden
    • none; Das Element kann von BEIDEN Seite überdeckt werden

Das "flex-layout

Das praktische am flex-layout ist, dass es wie der Name schon sagt flexibel auf Veränderung der Fenstergröße reagiert. Ein häufiger Anwendungsfall ist die Navigationsleiste einer Seite oder eine "Sidebar".

Dieses Layout besteht immer aus einem Eltern-Element (Wrapper) und aus Kind-Elementen (Items). Für beide gibt es Optionen welche verschiedene "Muster" bewirken. Die Eigenschaften welche unter "Für den Wrapper" aufgelistet sind, funktionieren logischerweise auch nur im Element welches als Wrapper genutzt wird. Für die Eigenschaften unter "Für die jeweiligen Items" gilt das gleiche.

welche Eigenschaften werden gebraucht?

    Für den Wrapper

  • display:
    • Welche Werte bewirken was?
    • flex; Legt das flex-Layout fest
  • flex-direction:
    • Welche Werte bewirken was?
    • row; Items fließen von Links nach Rechts
    • row-reverse; Items fließen von Rechts nach Links
    • column; Items fließen von Oben nach Unten
    • column-reverse; Items fließen von Unten nach Oben
  • flex-wrap:
    • Welche Werte bewirken was?
    • nowrap; (Standard) alle Items sind in einer Reihe
    • wrap; Die Items werden in mehrere Zeile geteilt wenn kein Platz mehr ist
    • wrap-reverse; Die Items werden in mehrere Zeile geteilt wenn kein Platz mehr ist und fließen von unten nach oben
  • flex-flow:
    • Welche Werte bewirken was?
    • z.B. row wrap; Diese Eigenschaft fasst nur flex-direction & flex-wrap in einen zusammen
  • justify-items:
    • Welche Werte bewirken was?
    • flex-start; (Standard) Items werden vom Start des Wrappers aus ausgerichtet
    • flex-end; Items werden vom Ende des Wrappers aus ausgerichtet
    • center; Items werden in der Mitte des Wrappers ausgerichtet
    • space-between; Zwischen den Items herscht ein gleichmäßiger Freiraum
    • space-around; Um die Items herum herscht ein gleichmäßiger Freiraum
    • space-evenly; Überall herscht ein gleichmäßiger Freiraum
  • align-items:
    • Welche Werte bewirken was?
    • stretch; (Standard) Alle Items nehmen die maximal verfügbare Größe an
    • flex-start; Alle Items werden am oberen Ende des Wrappers ausgerichtet
    • flex-end; Alle Items werden am unteren Ende des Wrappers ausgerichtet
    • center; Alle Items werden genau auf der Mitte des Wrappers ausgerichtet
    • baseline; Der Text der Items wird auf der Grundlinie des Wrappers ausgerichtet
  • align-content:
    • Welche Werte bewirken was?
    • Diese Eigenschaft funktioniert nur wenn es mehr als eine Reihe Items gibt
    • stretch; (Standard) Die Item-Reihen füllen den gesamt verfügbaren horizontalen Freiraum aus
    • flex-start; Die Item-Reihen werden alle am oberen Ende des Wrappers ausgerichtet
    • flex-end; Die Item-Reihen werden alle am unteren Ende des Wrappers ausgerichtet
    • center; Die Item-Reihen werden in der Mitte des Wrappers ausgerichtet
    • space-between; Zwischen den Item-Reihen wird gleichmäßiger Freiraum gelassen
    • space-around; Um die Item-Reihen wird ein gleichmäßiger Freiraum gelassen

    • Für die jeweiligen Items

  • order:
    • Mit einer Ganzzahl kann man die Reihenfolge des jeweiligen Items festlegen
  • flex-grow:
    • Gibt dem Item die Möglichkeit größer als die anderen zu werden falls nötig (Je größer die angegebene Zahl desto mehr Priorität)
  • flex-shrink:
    • Gibt dem Item die Möglichkeit kleiner als die anderen zu werden falls nötig (Je größer die angegebene Zahl desto mehr Priorität)
  • flex-basis:
    • Alle Einheiten sind erlaubt, setzt eine Grundgröße für das jeweilige Item
  • align-self:
    • Hier sind alle Werte welche auch bei align-items erlaubt sind erlaubt um das jeweilige Item separat zu ändern

Das "grid-layout"

Das Grid-Layout ist das umfangreichste Layout und wird hauptsächlich genutzt um das Gesamtlayout einer Seite festzulegen. Man kann nämlich mit diesem Layout sehr gut die einzelnen Elemente anordnen, in diesen selbst wird dann wiederrum oft das flex-layout verwendet.

Dieses Layout besteht wie das flex-layout immer aus einem Eltern-Element (Wrapper) und aus Kind-Elementen (Items). Für beide gibt es Optionen welche verschiedene "Muster" bewirken. Die Eigenschaften welche unter "Für den Wrapper" aufgelistet sind, funktionieren logischerweise auch nur im Element welches als Wrapper genutzt wird. Für die Eigenschaften unter "Für die jeweiligen Items" gilt das gleiche.

welche Eigenschaften werden gebraucht?

    Für den Wrapper

  • display:
    • Welche Werte bewirken was?
    • grid; Legt das grid-Layout fest
  • grid-template-columns:
    • 10% 50% 40%; Legt die Anzahl und Breite der Spalten fest (Jeder Wert repräsentiert eine Spalte, in diesem Fall 3 Spalten)
  • grid-template-rows:
    • 20% 40% 20% 10% 10%; Legt die Anzahl und Breite der Reihen fest (Jeder Wert repräsentiert eine Reihe, in diesem Fall 5 Reihen)
  • grid-template-areas:
    • "header header header"
      "content content sidebar"
      "content content sidebar"
      "content content sidebar"
      "footer footer footer";
      Dieser Aufbau ist nur ein Beispiel und kann beliebig sein. Wichtig ist, dass vorher auch genauso viele Spalten und Reihen definiert wurden
      (in diesem Fall 3 Spalten & 5 Reihen, siehe: grid-template-columns: & grid-template-rows:)

    • Für die jeweiligen Items

  • gird-column-start:
    • Mit einer Ganzzahl kann festlegt werden in welcher Spalte ein Item starten soll (z.B. 1 = Item startet in der ersten Spalte)
  • gird-column-end:
    • Mit einer Ganzzahl kann festlegt werden in welcher Spalte ein Item enden soll (z.B. 3 = Item endet in der dritten Spalte)
  • gird-row-start:
    • Mit einer Ganzzahl kann festlegt werden in welcher Reihe ein Item starten soll (z.B. 2 = Item startet in der zweiten Reihe)
  • gird-row-end:
    • Mit einer Ganzzahl kann festlegt werden in welcher Reihe ein Item enden soll (z.B. 4 = Item endet in der vierten Reihe)
  • grid-area:
    • Setzt grid-template-areas: im Wrapper vorraus, gibt dem Item seinen Platz (mögliche Werte wären header, content, sidebar oder footer)
